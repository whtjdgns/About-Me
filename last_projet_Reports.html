[Project] AI 기반 스마트 가계부 '핀로그' 기술 보고서

1. 개요 (Executive Summary)

    프로젝트명: 핀로그 (FinLog)

    목적: 거래 내역 입력 최소화 및 AI 자동 카테고리 분류를 통한 효율적인 개인 금융 관리 웹 애플리케이션 개발자로서의

    주요 기술: React.js, Node.js, Express.js, MySQL, Google Gemini API

    핵심 성과: 자연어 처리를 통한 거래 내역 자동 분류 기능 구현, 직관적인 대시보드 및 소비 리포트 제공

2. 서론 (Introduction)

2.1. 배경 및 필요성

기존 수기 가계부는 번거로운 입력과 카테고리 분류의 어려움으로 인해 지속적인 관리가 힘들다는 단점이 있었습니다. 개인 금융 데이터의 체계적 관리와 소비 습관 파악의 중요성이 증대됨에 따라, 생성형 AI 기술을 활용하여 사용자 경험을 혁신할 기회를 포착했습니다.

2.2. 프로젝트 목표

    자동화: 거래 내용 입력 시 AI가 적절한 카테고리를 즉시 추천하여 입력 시간 단축.

    시각화: 수입/지출 현황을 한눈에 파악할 수 있는 직관적인 대시보드 및 차트 제공.

    개인화: 사용자 정의 카테고리 기능을 통한 맞춤형 관리 환경 제공.

    보안성: JWT 기반의 안전한 사용자 인증 시스템 구축.

3. 시스템 설계 및 아키텍처

3.1. 기술 스택 (Tech Stack)

구분 기술 (Technology) 설명
Frontend React.js, Tailwind CSS 사용자 인터페이스 구현, SPA 구조, 반응형 디자인
Backend Node.js, Express.js RESTful API 서버 구축, 비즈니스 로직 처리
Database MySQL 관계형 데이터베이스, 사용자 및 거래 데이터 저장
AI/External Google Gemini API, Nodemailer 카테고리 추천 기능 수행, 이메일 발송 서비스
Auth JWT (JSON Web Token) 상태 비저장(Stateless) 방식의 안전한 인증 처리

3.2. 시스템 아키텍처 흐름

1 Client: 거래 내역 입력 (디바운싱 적용) -> API 요청
2 Server: 입력 내용 + 카테고리 후보군 패키징 -> AI API 요청
3 AI Engine: Google Gemini가 문맥 분석 후 최적 카테고리 반환
4 DB: 최종 거래 내역 저장 (Users, Transactions)
5 Client: 대시보드 및 차트 실시간 업데이트

3.3. 데이터베이스 설계 (ERD)

    Users: 사용자 인증 정보 및 프로필 저장.

    Transactions: 수입/지출 내역, 금액, 날짜 등 핵심 데이터.

    Categories: 시스템 기본 카테고리 및 사용자 정의 카테고리.

4. 핵심 기능 구현 (Key Feature Implementation)

4.1. AI 기반 카테고리 자동 추천 (핵심)

    작동 원리: 사용자가 "스타벅스 아메리카노" 입력 시, 백엔드에서 Gemini API에 최적화된 프롬프트(내용 + 후보군)를 전송합니다.

    최적화: 디바운싱(Debouncing) 기술을 적용하여 불필요한 API 호출을 최소화하고 비용을 절감했습니다.

    기술적 챌린지: AI 응답 지연 시간을 고려한 UI 처리 및 정확도 높은 프롬프트 엔지니어링 수행.

4.2. JWT 인증 및 보안

  1. 회원가입 시 비밀번호를 bcrypt로 해싱하여 안전하게 저장합니다.
  2. 로그인 성공 시 Access Token을 발급하고, API 요청 시 미들웨어를 통해 유효성을 검증합니다.

4.3. 대시보드 및 데이터 시각화

  1. 월별 수입/지출 합계를 실시간으로 계산하여 표시합니다.
  2. Chart.js를 활용하여 카테고리별 지출 비중을 파이 차트로 시각화했습니다.

5. 결과 및 성과 (Results & Outcomes)

5.1. 프로젝트 성과

    정성적 성과: 거래 내역 입력 시간을 획기적으로 단축하여 사용자 편의성 증대.

    기술적 성과: React, Node.js 환경에서 생성형 AI API를 실제 서비스 로직에 통합하는 역량 입증.

    차별화: 단순 기록용 가계부를 넘어선 지능형 금융 비서로의 발전 가능성 확인.

5.2. 개발 과정의 이슈 및 해결 (Troubleshooting)

Issue 1: 비동기 데이터 처리 이슈

문제: 대시보드 로딩 시 데이터가 제때 표시되지 않거나 무한 루프 발생.

해결: useEffect의 의존성 배열(Dependency Array)을 정확히 관리하고, 데이터 Fetching 로직을 커스텀 훅으로 분리하여 상태 관리 최적화.

Issue 2: 배포 리소스 관리

문제: AWS 배포 과정에서 RDS 등 클라우드 비용 및 리소스 관리의 어려움.

해결: 프리 티어 리소스 한도 내에서 최적의 인스턴스 타입을 선정하고, 불필요한 연결을 정리하는 Connection Pool 설정 최적화.
